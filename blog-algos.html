Mémoriser les algorithmes de tri : ma méthode visuelle

Bubble sort, merge sort, quicksort... Comment j'ai arrêté de les confondre grâce à des images mentales simples et des analogies concrètes.

Rayan Remili
5 janvier 2025
6 min de lecture

En première année à l'ESTIN, on nous a demandé d'implémenter une dizaine d'algorithmes de tri en C. Le problème : ils se ressemblaient tous dans ma tête. Je confondais le bubble sort et le selection sort, je mélangeais merge sort et quicksort. Voici la méthode que j'ai trouvée pour ne plus jamais les confondre.

L'idée : associer chaque algo à une image

Au lieu d'essayer de mémoriser la définition formelle ou le code, j'ai associé chaque algorithme à une scène concrète du quotidien. Quand je pense à l'algo, l'image vient d'abord — et le code suit naturellement.

🧠 Notre mémoire retient les images 6x mieux que les mots abstraits. Exploite ça.

Les 4 algorithmes essentiels

Bubble Sort — les bulles qui remontent
⏱ O(n²) 💾 O(1) 📊 Simple
Image : des bulles dans un verre d'eau qui remontent une par une vers la surface. À chaque passage, le plus grand élément "remonte" à la fin du tableau. Tu compares deux voisins, tu les échanges si besoin, tu avances. C'est lent, mais c'est le plus facile à visualiser et à coder.
Selection Sort — chercher le minimum à chaque fois
⏱ O(n²) 💾 O(1) 📊 Simple
Image : tu tries une main de cartes en cherchant à chaque fois la plus petite carte dans le tas restant, et tu la places en premier. Simple et intuitif, mais autant de comparaisons que le bubble sort.
Merge Sort — diviser pour régner
⏱ O(n log n) 💾 O(n) 📊 Efficace
Image : tu prends une pile de feuilles, tu la coupes en deux, tu retries chaque moitié, puis tu fusionnes les deux moitiés triées. Récursif, efficace, mais nécessite de la mémoire supplémentaire pour stocker les sous-tableaux.
Quicksort — choisir un pivot et partitionner
⏱ O(n log n) moy. 💾 O(log n) 📊 Très efficace
Image : tu choisis quelqu'un dans la salle comme référence (le pivot), tu mets tout le monde plus petit à gauche et plus grand à droite, puis tu recommences dans chaque groupe. En pratique, le plus rapide des quatre — c'est celui que tu verras le plus en vrai.

Le code en C pour fixer les idées

Voici le bubble sort — le plus simple — pour voir comment l'image se traduit en code :

void bubbleSort(int arr[], int n) { for (int i = 0; i < n-1; i++) { for (int j = 0; j < n-i-1; j++) { if (arr[j] > arr[j+1]) { // Échange — la bulle remonte int temp = arr[j]; arr[j] = arr[j+1]; arr[j+1] = temp; } } } }

Comment retenir lequel utiliser ?

  • Petit tableau (< 50 éléments) → bubble ou selection sort. La simplicité prime.
  • Grand tableau, mémoire disponible → merge sort. Stable et prévisible.
  • Grand tableau, performance maximale → quicksort. Le choix de la plupart des librairies standard.
  • Données presque triées → insertion sort (le bonus). Il est O(n) dans ce cas.

La vraie leçon

Ces algorithmes ne sont pas juste des exercices académiques. Comprendre comment ils fonctionnent m'a aidé à écrire du code plus efficace au quotidien — savoir quand utiliser une structure triée, quand un O(n²) est acceptable, et quand il faut absolument optimiser.

Et pour les révisions d'exam : une feuille A4 avec les images mentales + les complexités. C'est tout ce qu'il faut.
