CSS Grid vs Flexbox : quand utiliser quoi concrètement

Après des dizaines de projets, voici ma règle simple pour ne plus jamais hésiter entre les deux.

Rayan Remili
18 décembre 2024
5 min de lecture

Quand j'ai appris le CSS, j'utilisais Flexbox pour tout. Absolument tout. Puis j'ai découvert Grid et je suis passé à l'extrême inverse. Aujourd'hui, après avoir construit une dizaine de projets, j'ai une règle claire que j'applique sans me poser de question.

La règle en une phrase

Flexbox = une direction. Grid = deux directions.

C'est aussi simple que ça. Si tu disposes des éléments dans une seule ligne ou une seule colonne, Flexbox. Si tu construis un vrai layout en lignes ET colonnes simultanément, Grid.

💡 Flexbox pense en "comment je distribue de l'espace dans une ligne ?" Grid pense en "comment je divise mon espace en zones ?"

✓ Flexbox — utilise pour...
  • Une navbar avec logo + liens + bouton
  • Des boutons côte à côte
  • Centrer un élément verticalement
  • Une liste de tags/badges
  • Footer avec 3 colonnes simples
  • Aligner icône + texte
✓ Grid — utilise pour...
  • Le layout général de la page
  • Une grille de cartes produits
  • Un dashboard avec sidebar
  • Une galerie photo responsive
  • Tout ce qui ressemble à un tableau
  • Hero en deux colonnes

Exemples concrets tirés de mes projets

La navbar de mon portfolio — Flexbox

Logo à gauche, liens au centre, bouton CV à droite. C'est une seule ligne horizontale avec de l'espacement entre les éléments.

header { display: flex; align-items: center; justify-content: space-between; padding: 0 32px; height: 56px; }

La grille de projets — Grid

Des cartes disposées en 2 colonnes qui s'adaptent automatiquement à la taille de l'écran. C'est exactement le cas d'usage de Grid.

.projects-grid { display: grid; grid-template-columns: 1fr 1fr; gap: 12px; } /* Responsive automatique */ @media (max-width: 700px) { .projects-grid { grid-template-columns: 1fr; } }

Le hero en deux colonnes — Grid

Texte à gauche, carte profil à droite — deux zones distinctes, donc Grid.

.hero { display: grid; grid-template-columns: 1fr 300px; gap: 40px; align-items: center; }

Est-ce qu'on peut mélanger les deux ?

Oui, et c'est même recommandé. Grid pour le layout global de la page, Flexbox pour les composants à l'intérieur. Dans mon portfolio, le layout général est en Grid, mais les cartes projet en elles-mêmes utilisent Flexbox pour aligner leur contenu.

La règle finale

  • Tu veux aligner des choses sur un axe → Flexbox
  • Tu veux créer un layout en zones → Grid
  • Tu n'es pas sûr → essaie Flexbox d'abord, si ça se complique, passe à Grid
