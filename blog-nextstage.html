Comment j'ai construit NextStage en 2 semaines sans framework

Le défi : créer une plateforme complète d'accompagnement pour les étudiants algériens, de zéro, avec uniquement HTML, CSS et JavaScript vanilla.

Rayan Remili
15 mars 2025
5 min de lecture

Le contexte

Tout a commencé en septembre 2023, quand j'ai réalisé que beaucoup d'étudiants algériens autour de moi galeraient avec les démarches Campus France. La procédure est complexe, les infos éparpillées partout, et les erreurs coûtent cher — parfois une année entière de retard.

J'avais deux semaines avant la rentrée universitaire. Pas assez de temps pour apprendre React, configurer un backend, ou réfléchir à une base de données. La contrainte était claire : HTML, CSS, JS pur. Rien d'autre.

💡 La contrainte technique est souvent une bénédiction déguisée. Elle force à aller à l'essentiel et à livrer plutôt que de peaufiner indéfiniment.

Les décisions clés

1. Pas de framework, pas de bundler

La première tentation a été d'utiliser React ou Vue. Mais j'ai vite compris que le temps de setup, de configuration et d'apprentissage aurait mangé mes deux semaines. J'ai choisi vanilla JS avec des modules import/export natifs du navigateur — supportés par tous les navigateurs modernes, sans configuration.

2. Un seul fichier CSS avec des variables partout

Au lieu de créer 15 fichiers CSS, tout est dans un seul fichier avec des variables CSS. Ça paraît basique, mais ça m'a sauvé des heures de débogage. Changer la couleur principale ? Une seule ligne à modifier.

:root { --primary: #5263ff; --bg: #0f0f0f; --text: rgba(255,255,255,0.85); }

3. Le contenu d'abord, le design ensuite

Erreur classique que j'évite maintenant : passer des heures sur les animations avant d'avoir le contenu. J'ai d'abord écrit tout le texte des sections — TCF, Pastel, universités, visa — puis j'ai designé autour du contenu. Pas l'inverse.

Ce que j'ai appris

  • Le responsive mobile d'abord — la majorité des étudiants visitent depuis leur téléphone. Commencer par le mobile m'a évité de tout refaire.
  • FormSubmit pour les formulaires — pas besoin de backend pour recevoir les messages de contact. Un simple action="https://formsubmit.co/..." suffit.
  • Vercel pour le déploiement — de zéro à en ligne en moins de 3 minutes. Git push = site mis à jour automatiquement.
  • Tester sur son vrai téléphone à chaque étape — l'émulateur du navigateur ne remplace pas un vrai appareil. Les espacements, les tailles de police, tout se comporte différemment.

Le résultat

NextStage est en ligne depuis mars 2023. Le site a été visité par plusieurs centaines d'étudiants algériens. J'ai reçu des messages de personnes qui ont réussi leur procédure Campus France grâce aux ressources disponibles.

Est-ce que j'aurais dû utiliser un framework pour la maintenabilité à long terme ? Probablement. Mais pour lancer rapidement et valider que le projet avait de la valeur auprès de vrais utilisateurs — non. Shipped is better than perfect.
